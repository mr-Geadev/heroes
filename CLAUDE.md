# Board Strategy Game — Prototype Spec (Claude Code)

## Обзор

Минимально рабочий прототип пошаговой мультиплеерной стратегии на зацикленном поле. **Лютый MVP** — без спрайтов, всё схематически/текстом. Цель: потыкать, как оно играется.

**Стек:** Angular (latest stable) + Node.js WebSocket-сервер (ws library).
**Платформа:** Десктоп (браузер), никакого мобильного адаптива.
**Игроки:** Всегда 4. Каждый игрок — отдельная вкладка/устройство. Играть с одной вкладки нельзя.

---

## Архитектура

### Монорепо
```
/
├── server/          # Node.js + ws
│   ├── index.ts
│   ├── game-state.ts
│   ├── types.ts
│   └── ...
├── client/          # Angular app
│   └── src/
└── package.json     # workspace root
```

### Принцип разделения
- **Вся игровая логика на клиенте.** Клиент считает бои, доход, результаты событий.
- **Сервер — тупой ретранслятор.** Принимает действия от игрока, рассылает всем остальным. Хранит текущий GameState как зеркало, но не валидирует.
- Один WebSocket-сервер, клиент Angular.

### Сетевая модель
1. Хост создаёт комнату → сервер возвращает 6-символьный код (буквы + цифры).
2. Другие игроки вводят код + своё имя → подключаются к комнате.
3. Хост видит список подключённых, жмёт "Начать игру".
4. Сервер рассылает `game:start` с начальным состоянием, сгенерированным на клиенте хоста.
5. Во время игры: клиент текущего игрока отправляет действие → сервер рассылает → все клиенты обновляют состояние.

### WebSocket-сообщения (примерный формат)
```typescript
// Client → Server
{ type: 'room:create', payload: { playerName: string } }
{ type: 'room:join', payload: { code: string, playerName: string } }
{ type: 'room:start' }  // только хост
{ type: 'game:action', payload: GameAction }

// Server → Client
{ type: 'room:created', payload: { code: string } }
{ type: 'room:player_joined', payload: { players: Player[] } }
{ type: 'room:error', payload: { message: string } }
{ type: 'game:start', payload: { state: GameState } }
{ type: 'game:update', payload: GameAction }  // ретрансляция
```

## Игровое поле

### Генерация
- **28 клеток** по кругу (квадрат 8×8 — по 7 на сторону, как монополия).
- Генерируется случайно, но с паттерном:
    - На каждой стороне ровно 2 источника ресурсов (итого 8).
    - На каждой стороне ровно 1 данж (итого 4).
    - Остальные клетки — случайные события.
    - Угловые клетки (4 шт) — стартовые точки игроков (не считаются ресурсами/данжами/событиями, просто пустые).

### Типы клеток
| Тип | Количество | Описание |
|-----|-----------|----------|
| `start` | 4 | Стартовые углы |
| `resource` | 8 | Источники ресурсов (случайный тип из 3) |
| `dungeon` | 4 | Данжи с охраной и наградой |
| `event` | 12 | Случайные события |

### Визуализация
- Квадратное поле, клетки по периметру (как Монополия).
- Каждая клетка — прямоугольник с:
    - Иконкой типа (emoji или буква: ⛏️ R / 🏰 D / ❓ E / 🏁 S)
    - Владелец (цветная рамка если захвачена)
    - Фишки игроков на клетке (цветные кружки)
- Центр поля — информационная панель (текущий ход, счёт, текущая фаза).

## Ресурсы

Три типа:
| Ресурс | Emoji | Цвет |
|--------|-------|------|
| Дерево | 🪵 | Коричневый |
| Камень | 🪨 | Серый |
| Золото | 💰 | Жёлтый |

### Доход
- Каждый захваченный источник приносит **2 единицы** своего ресурса в начале хода владельца.
- Базовый доход: **1 дерево, 1 камень, 1 золото** каждый ход (гарантированный минимум для всех).

## Юниты (5 штук)

Простая боевка: только HP и урон. Без брони, спецспособностей, типов (ближний/дальний/поддержка) — всё это для MVP не нужно.

| Юнит | HP | Урон | Стоимость |
|------|----|------|-----------|
| Ополченец | 5 | 2 | 2🪵 |
| Лучник | 3 | 3 | 2🪨 |
| Рыцарь | 8 | 3 | 2🪵 2🪨 |
| Маг | 4 | 5 | 3💰 |
| Голем | 12 | 2 | 2🪨 2💰 |

### Пул найма
- Общий пул из **4 юнитов**, видимый всем.
- Покупка: **один юнит за ход**, в любой момент хода (кроме боя).
- Купленный юнит добавляется **в конец армии** (без расстановки).
- На месте купленного мгновенно появляется новый случайный юнит.

---

## Армия и бой

### Армия
- Максимум **5 юнитов** в армии.
- Расстановки нет — юниты просто занимают слоты 1-5 в порядке найма.

### Автобой
1. Юнит в слоте 1 атакует юнит в слоте 1 противника. Оба получают урон (одновременная атака).
2. Если противоположный слот пуст — атака переходит на следующий занятый.
3. Раунд = все пары обменялись ударами.
4. Мёртвые юниты (HP ≤ 0) убираются, живые сдвигаются к началу.
5. Повторяется до уничтожения одной из армий.
6. Максимум 20 раундов (защита от бесконечного цикла) — если никто не выиграл, ничья (обе армии уничтожаются).

### Бой с охраной данжа/ресурса
- У нейтральной охраны тоже есть армия (1-3 юнита, генерируются при создании поля).
- Те же правила боя.

### Совместный данж
- Игрок на данже может пригласить одного другого игрока. Одна кнопка "Пригласить" → выбор игрока → тот получает уведомление и может принять/отклонить.
- При совместном бое армии объединяются (до 10 юнитов против охраны).
- После победы пригласивший распределяет награду. Артефакты неделимы.

---

## Артефакты (5 штук)

Находятся в данжах. Каждый артефакт занимает свой слот. Нельзя надеть два предмета одного слота.

| Артефакт | Слот | Эффект |
|----------|------|--------|
| Кольчуга | Одежда | Все юниты армии получают +2 HP |
| Огненный жезл | Руки | Все юниты армии получают +1 урон |
| Кольцо удачи | Кольцо | +1 к результату броска кубика |
| Медальон мудреца | Медальон | Доход с источников ×1.5 (округление вверх) |
| Походный рюкзак | Доп. слот | Можно нанимать 2 юнита за ход |

### Слоты артефактов
Одежда (1), Руки (1), Кольцо (1), Медальон (1), Доп. слот 1 (1), Доп. слот 2 (1).
Для MVP — всего 5 артефактов, каждый в своём слоте (доп. слот 2 пока пустует).

---

## Случайные события (5 штук)

При попадании на клетку `event` — случайно выбирается одно:

| Событие | Эффект |
|---------|--------|
| Попутный ветер | Дополнительное перемещение на 3 клетки вперёд (без срабатывания клетки) |
| Налёт разбойников | Потеря 2 случайных ресурсов |
| Находка | Получить 3 единицы случайного ресурса |
| Дезертир | Потеря последнего юнита в армии (если армия пуста — ничего) |
| Благословение | +2 очка победы |

## Торговля

- В фазу торговли (конец хода) игрок может отправить **одно предложение** любому другому игроку.
- В сделке участвуют: ресурсы, территории (без гарнизона), артефакты, обещания.
- UI: простой модальный диалог. Выбираешь что отдаёшь, что хочешь. Второй игрок видит предложение, жмёт "Принять" или "Отклонить".

### Обещания
- Можно добавить к сделке обещание (текстовый ввод, например "Не нападать 3 хода").
- Обещания **не обязательны** к исполнению.
- У каждого игрока видна статистика: "Выполнил X из Y обещаний".
- В MVP обещания — просто текст. Никакой автоматической проверки выполнения.

## Структура хода

```
1. ДОХОД
   → Базовый: +1 🪵, +1 🪨, +1 💰
   → Со своих источников: +2 каждого типа за точку
   → Модификатор медальона если есть

2. НАЙМ (опционально)
   → Показать пул из 4 юнитов
   → Можно купить 1 (или 2 с рюкзаком)
   → Юнит добавляется в конец армии
   → Замена в пуле

3. БРОСОК КУБИКА
   → 2d6, перемещение
   → Модификатор кольца если есть

4. ДЕЙСТВИЕ НА КЛЕТКЕ
   → resource: бой с охраной или пропуск
   → dungeon: бой (одиночный или совместный) или пропуск
   → event: случайное событие
   → start: ничего

5. ТОРГОВЛЯ (опционально)
   → Одно предложение другому игроку
   → Принять / Отклонить
```

### Между ходами
- Пока ходит другой игрок, ты можешь получить:
    - Предложение обмена (принять / отклонить)
    - Приглашение в данж (принять / отклонить)

## Условие победы

- **Первый до 20 очков** — мгновенная победа.
- **Через 30 ходов** (каждый игрок сходил ~7-8 раз) — побеждает игрок с наибольшим количеством очков. При ничьей — у кого больше ресурсов.
- Игрок может **сдаться** в любой момент — его точки становятся нейтральными, армия распускается.

### Источники очков
| Действие | Очки |
|----------|------|
| Захват источника ресурсов | +1 |
| Победа в данже | +3 |
| Победа над игроком | +2 |
| Поражение от игрока | -1 |
| Событие "Благословение" | +2 |

## Детали для разработки

### GameState (объект состояния)
```typescript
interface GameState {
  board: Cell[];                // 28 клеток
  players: Player[];            // 4 игрока
  currentPlayerIndex: number;
  phase: 'income' | 'hire' | 'roll' | 'action' | 'trade' | 'waiting';
  turnNumber: number;           // общий счётчик ходов
  hirePool: Unit[];             // 4 юнита в пуле
  maxTurns: number;             // 30
  winScore: number;             // 20
  log: string[];                // лог событий
}

interface Player {
  id: string;
  name: string;
  position: number;             // индекс клетки
  resources: { wood: number; stone: number; gold: number };
  army: Unit[];                 // макс 5
  artifacts: Artifact[];
  score: number;
  promises: Promise[];          // данные обещания
  promisesKept: number;
  promisesBroken: number;
  surrendered: boolean;
  color: string;
}

interface Cell {
  index: number;
  type: 'start' | 'resource' | 'dungeon' | 'event';
  resourceType?: 'wood' | 'stone' | 'gold';
  owner?: string;               // player id
  garrison?: Unit[];            // охрана (нейтральная или гарнизон)
  dungeonReward?: DungeonReward;
  dungeonCleared?: boolean;
}

interface Unit {
  id: string;
  name: string;
  hp: number;
  maxHp: number;
  damage: number;
  cost: { wood: number; stone: number; gold: number };
}

interface Artifact {
  id: string;
  name: string;
  slot: 'armor' | 'weapon' | 'ring' | 'medallion' | 'extra1' | 'extra2';
  effect: string;               // описание для UI
}

interface DungeonReward {
  resources: { wood: number; stone: number; gold: number };
  artifact?: Artifact;
  score: number;
}

interface Promise {
  text: string;
  from: string;                 // player id
  to: string;                   // player id
  turnCreated: number;
  fulfilled?: boolean;
}
```

### Генерация охраны данжа
- Лёгкий данж: 1 юнит (случайный, но слабый — ополченец или лучник).
- Средний данж: 2 юнита (рыцарь + ополченец).
- Тяжёлый данж: 3 юнита (рыцарь + маг + голем).
- Равномерно распределить по 4 данжам: 1 лёгкий, 2 средних, 1 тяжёлый.

### Генерация награды данжа
- Лёгкий: 3 случайных ресурса + 1 очко.
- Средний: 5 случайных ресурсов + 2 очка.
- Тяжёлый: 5 случайных ресурсов + 3 очка + артефакт (случайный из тех, что ещё не разыграны).

### Генерация охраны источника ресурсов
- 1 юнит: случайный ополченец или лучник. После захвата — охраны нет (пока игрок не поставит гарнизон).
- Игрок может оставить юнита из армии как гарнизон (одна кнопка: "Оставить последнего юнита").

## Что НЕ делать

- Никаких спрайтов, картинок, анимаций (кроме минимальных CSS-переходов).
- Никакой авторизации, базы данных, персистентности.
- Никакого мобильного адаптива.
- Никакой валидации на сервере (клиент — авторитетный).
- Никаких типов юнитов (ближний/дальний/поддержка) — просто HP и урон.
- Никакой расстановки армии — юниты всегда в порядке найма.
- Никакого AI для ботов.
- Не нужно красиво — нужно функционально.

## Порядок разработки (рекомендация)

1. **Сервер:** комнаты, подключение, ретрансляция сообщений.
2. **Лобби:** создание комнаты, подключение, старт игры.
3. **Поле:** генерация, визуализация квадрата клеток.
4. **Ход:** доход → найм → бросок → перемещение → действие → торговля.
5. **Бой:** автоматический, отображение результата.
6. **Данжи:** охрана, награда, совместные.
7. **Торговля и обещания:** модальный диалог.
8. **Победа:** проверка условий, экран результата.
