# Board Strategy Game — Spec

## Терминология

| Термин | Определение |
|--|--|
| **Текущий игрок** | Тот, за кого играет пользователь в данной вкладке (независимо от того, чей сейчас ход) |
| **Другие игроки** | Все игроки, кроме текущего |
| **Активный игрок** | Тот, чей сейчас ход |
| **Юнит** | Один отдельный боец любого типа |
| **Отряд** | Несколько одинаковых юнитов в одном слоте (`ArmySlot`) |
| **Армия** | Совокупность отрядов у героя или локации |

---

## Обзор
MVP пошаговой мультиплеерной стратегии. Без спрайтов, всё схематично. 2–4 игрока, каждый — отдельная вкладка.

**Стек:** Angular + Node.js (ws). Десктоп только.

---

## Архитектура
Монорепо: `/server/` (index.ts, room-manager.ts, message-handler.ts, types.ts) + `/client/` (Angular).

- **Вся логика на клиенте.** Сервер — ретранслятор, хранит `gameState` зеркально (для восстановления сессии), не валидирует.
- Хост создаёт комнату → 6-символьный код → остальные подключаются → хост жмёт «Начать».
- Сервер рассылает `game:start` с состоянием от клиента хоста.

**Сообщения:**
```
Client→Server: room:create, room:join, room:rejoin, room:start, game:action
Server→Client: room:created, room:self, room:player_joined, room:player_left, room:error, game:start, game:update
```

`game:action` payload всегда `{ type: 'state:update', payload: GameState }`.
При `room:rejoin` сервер отправляет текущий `game:update` с сохранённым стейтом до `room:player_joined`.

---

## Восстановление сессии

- Клиент сохраняет в `sessionStorage`: `roomCode`, `playerId`, `playerName`, `isHost`, `players`, `gameState`, `route`.
- При обновлении страницы: восстанавливает данные из sessionStorage, отправляет `room:rejoin`, навигирует на сохранённый маршрут.
- Сервер при `room:rejoin`: находит игрока по `playerId`, обновляет его WebSocket, снимает флаг `disconnected`, отправляет актуальный `gameState`, затем рассылает `room:player_joined`.
- Игроки **не удаляются** из комнаты при дисконнекте — помечаются `disconnected: true`, WS зануляется. `broadcast` молча пропускает их.
- Комната сохраняется даже если все игроки отключились (нет удаления при пустой комнате).

## Поле
28 клеток по кругу (8×8 квадрат, по периметру). Генерируется случайно:
- 4 угла — города (`start`), индексы 0, 7, 14, 21
- На каждой стороне (6 клеток): 2 источника ресурсов, 2 данжа, 2 события → итого 8 `resource`, 8 `dungeon`, 8 `event`

Клетка: иконка типа (🪵/🪨/💰/⚔️/❓/🏰), цветная рамка владельца, фишки игроков (цветные кружки).
Захваченная клетка — полупрозрачный фон цвета владельца.

**Топонимы:** каждой ресурсной клетке при генерации присваивается уникальное фэнтезийное имя вида «Шахта Марибор», «Лесопилка Туманная» и т.д. Пул из 30 топонимов, 8 выбираются случайно без повторений.

**Центральная панель поля:**
- Очки всех игроков + номер хода
- Пул найма (4 слота в сетке 2×2) — виден всем, кликабелен только в свой ход вне `combat` и `init_roll`
- Кнопка «↻ обновить пул» — стоит 5🪵 5🪨 5💰, доступна в свой ход вне `combat` и `init_roll`

## Справочник: Ресурсы

| | Emoji | Цвет | Источник |
|--|--|--|--|
| Дерево | 🪵 | Коричневый | Лесопилка |
| Камень | 🪨 | Серый | Шахта |
| Золото | 💰 | Жёлтый | Прииск |

Базовый доход: +1 каждого в начале хода. Захваченный источник: +2 своего типа.
Медальон мудреца: доход с источников ×1.5 (округление вверх), применяется до суммирования.

**Отображение в панели:** `10(+3)` — текущее количество и прирост за ход в скобках.

## Справочник: Маана

Числовой параметр героя (`GamePlayer.mana`). Максимум: `MAX_MANA = 10`.
Восстанавливается до максимума при прохождении через собственный город (включая остановку).
Механика расхода маны не реализована — поле зарезервировано.

## Справочник: Юниты

| Юнит | HP | Урон (мин–макс) | Стоимость | Макс в слоте | Класс | Особое |
|--|--|--|--|--|--|--|
| Ополченец | 5 | 1–3 | 2🪵 | 5 | ближний | — |
| Лучник | 3 | 2–4 | 2🪨 | 4 | дальний | — |
| Рыцарь | 8 | 2–4 | 2🪵 2🪨 | 2 | ближний | — |
| Маг | 4 | 3–7 | 3💰 | 2 | дальний | универсальный |
| Голем | 12 | 1–3 | 2🪨 2💰 | 1 | ближний | — |

**Классы боя:** юнит ближнего боя отвечает только на атаки ближних; юнит дальнего — только на атаки дальних. **Универсальный** (Маг) отвечает на любые атаки.

**Пул найма:** 4 слота `HireSlot[]`, отображаются в центре поля. Одна покупка (слота) за ход — доступна в любой фазе кроме `combat` и `init_roll`. Слот содержит 1–N одинаковых юнитов. Покупается целиком. Если нет свободного слота в армии и нет слота того же типа — покупка заблокирована. После покупки слот заменяется новым случайным.

**Бонусный найм:** при остановке в собственном городе игрок получает +1 найм (`hireBonusCount`). Бонус сохраняется до следующего хода и позволяет нанять второй отряд за ход.

**Обновление пула:** 5🪵 + 5🪨 + 5💰 → весь пул заменяется новыми случайными слотами.

## Справочник: Артефакты

| Артефакт | Слот | Эффект | Реализован |
|--|--|--|--|
| Кольчуга | `armor` | Все юниты +2 HP | ✅ |
| Огненный жезл | `weapon` | Все юниты +1 к damageMin и damageMax | ✅ |
| Кольцо удачи | `ring` | +1 к броску кубика | ✅ |
| Медальон мудреца | `medallion` | Доход с источников ×1.5 (ceil) | ✅ |
| Походный рюкзак | `extra1` | Нанимать 2 юнита за ход | ❌ не реализован |

Артефакты выдаются как награда тяжёлого данжа. Слот уже занятого типа не заменяется.

## Справочник: Случайные события

| id | Название | Эффект | Реализован |
|--|--|--|--|
| `wind` | 🌬 Попутный ветер | +3 клетки вперёд (клетка не срабатывает, только перемещение) | ✅ |
| `bandits` | 🗡 Налёт разбойников | −2 случайных ресурса | ✅ |
| `find` | 💎 Находка | +3 случайного ресурса | ✅ |
| `deserter` | 🏳 Дезертир | Потеря 1 юнита из последнего слота | ✅ |
| `blessing` | ✨ Благословение | +2 очка победы | ✅ |

Модалка события: фиксированное название сверху + динамический эффект снизу. Данные хранятся в `GameState.pendingEvent`. **Эффект применяется только после нажатия «Продолжить»** (`commitEvent`), не при попадании на клетку. После «Продолжить» → новая клетка обрабатывается (если ветер занёс на данж/источник — открывается соответствующая модалка).

## Справочник: Обещания

| Тип | Описание |
|--|--|
| `no-attack` | Не нападать 5 ходов |
| `help-battle` | Помочь в следующем бою |

Не обязательны к исполнению. Хранятся в `GamePlayer.promises`. Видны в панели игрока с указанием направления (→ кому / ← от кого). Автоматическое отслеживание не реализовано.

## Справочник: Данжи

| Сложность | Охрана | Награда | Кол-во на карте |
|--|--|--|--|
| Лёгкий | 3× Ополченец + 2× Лучник + 1× Рыцарь | 3 ресурса + 1 очко | 2 |
| Средний | 3× Ополченец + 2× Лучник + 2× Рыцарь + 1× Голем | 5 ресурсов + 2 очка | 4 |
| Тяжёлый | 5× Ополченец + 4× Лучник + 2× Рыцарь + 2× Мага + 1× Голем | 5 ресурсов + 3 очка + артефакт | 2 |

Распределение сложностей по карте: 2× лёгкий, 4× средний, 2× тяжёлый (перемешиваются случайно, по 2 на сторону).

В модалке данжа показывается охрана, **награда скрыта**.

**Регенерация:** после прохождения `dungeonCleared: true`. В начале хода следующего игрока разрушенные данжи перегенерируются с новой охраной и наградой.

**Охрана источника ресурсов:** случайная — 1–3× Ополченец + с вероятностью 60% ещё 1–2× Лучник. После захвата охрана исчезает.

---

## Справочник: Города

Стартовые клетки (индексы 0, 7, 14, 21) — города, принадлежащие игрокам.

**Иконка:** 🏰. Имя: «Город <Имя игрока>» или «Заброшенный замок».

**Расположение игроков по числу участников:**
- 4 игрока: позиции 0, 7, 14, 21
- 3 игрока: позиции 0, 7, 14; город 21 — заброшенный
- 2 игрока: позиции 0, 14 (через один угол); города 7 и 21 — заброшенные

**Заброшенный замок** — нет владельца, охрана: 5× Ополченец + 4× Лучник + 2× Рыцарь + 2× Мага + 1× Голем.

**Прохождение через свой город** (в т.ч. при остановке): армия полностью исцеляется (hp → maxHp), мана восстанавливается до `MAX_MANA`.

**Остановка в своём городе:** +1 бонусный найм (`hireBonusCount++`), который можно использовать в текущий или следующий ход.

**Остановка в чужом городе:** `phase = 'cell'` → модалка с кнопкой «Атаковать» или «Пропустить». Охрана города — `playerGarrison` владельца. Если охраны нет — мгновенный захват. После победы город переходит к атакующему (`cell.owner = p.id`).

**Поражение:** проигравший возвращается в ближайший собственный город (минимальное круговое расстояние). Если собственных городов нет — игрок выбывает (`surrender`).

## Армия и бой

Макс 5 **слотов** в армии. Одинаковые юниты стакаются в один слот (`ArmySlot { unit, count }`).

**Автобой:**
- Очерёдность: atk[0], def[0], atk[1], def[1], … (по слотам, по очереди)
- Урон стака = сумма индивидуальных бросков каждого юнита
- Входящий урон снимается с одного юнита: `totalHP = (count-1)*maxHp + frontHp`
- **Ответный удар:** юнит отвечает только если класс атакующего совпадает с его классом (или он универсальный). Каждый юнит отвечает не более одного раза за раунд.
- Макс 20 раундов. Ничья (оба выжили) → обе армии уничтожаются. Взаимное уничтожение → атакующие победили если клетка нейтральная, проиграли если клетка другого игрока.

**Визуализация боя:** модалка для всех с анимацией раундов (lunge + HP-бары). Бой 2-шаговый: `startCapture/startDungeon/startCityCapture` (phase=`combat`) → `resolveCombat` (phase=`free`). После победы в данже — попап с наградой.

**Токен:** анимируется пошагово через каждую клетку (180ms/клетка), видно всем. Модалка действия открывается только после завершения анимации.

**God mode** (`IDDQD()` в консоли): атакующие юниты не получают урона.

### Совместный бой

Активный игрок в `phase = 'cell'` может нажать «🤝 Помощь: <Имя>» → отправляется запрос (`phase = 'joint_request'`, `GameState.jointRequest`).

- Запрошенный союзник видит модалку с кнопками «✅ Принять» / «❌ Отклонить»
- Остальные видят «⏳ Ждём ответа...»
- **Принять (`acceptJoint`):** запускается совместный бой. Армии объединяются в 10 слотов (0–4 атакующий, 5–9 союзник)
- **Отклонить (`declineJoint`):** возврат в `phase = 'cell'`

**Очерёдность в совместном бою:** atk[0] → ally[0] → def[0] → atk[1] → ally[1] → def[1] → …

**Целевой выбор защитников:** при атаке защитника проверяется его слот i у атакующего и союзника. Оба живы → рандом. Один жив → атакует его. Никого → следующий слот.

**После боя:** выжившие юниты атакующего и союзника восстанавливаются в их армии. При поражении обе армии обнуляются.

## Торговля

Трейд доступен в любой фазе кроме `combat`. Одна активная сделка одновременно (`pendingOffer` в GameState). **Города нельзя передавать в торговле** (только ресурсные клетки).

**Модалка трейда — 4 колонки:**
```
[Мои вещи] | [Я даю] | [Я прошу] | [Вещи игрока]
```
- Клик по вещи в колонке 1 или 4 добавляет её в колонку предложения (2 или 3)
- В колонках предложения у ресурсов — слайдер выбора количества; у любого элемента — ✕ для удаления
- Обещания — полноценный товар в колонках 1 и 4, добавляются/убираются кликом
- Внизу: «Отправить» / «Отмена»

**Получение трейда:** та же модалка, колонки 2/3 read-only.
- Кнопки: «✓ Принять» / «↩ Встречное предложение» / «✕ Отклонить»

## Структура хода

```
0. INIT ROLL (однократно): все игроки бросают кубик для определения очерёдности
1. ДОХОД (авто): +1 каждого + с источников + медальон
2. БРОСОК 2d6 (модалка с анимацией) + кольцо удачи (+1)
   → При движении: если токен проходит через собственный город — армия исцеляется, мана восстанавливается
   → Найм и трейд доступны уже здесь
3. ДЕЙСТВИЕ НА КЛЕТКЕ (модалка, после анимации токена):
   - resource (чужая/нейтральная): бой, совместный бой или пропуск
   - resource (своя): мгновенный доп. доход (+2, с учётом медальона) → free
   - dungeon (не пройден): бой, совместный бой или пропуск; после победы — попап с наградой
   - dungeon (пройден/очищен): сразу free
   - event: показ названия + эффект ПОСЛЕ «Продолжить» → обработка новой клетки если нужно
   - start (свой город): исцеление + мана + бонусный найм → free
   - start (чужой город): атака или пропуск
4. JOINT_REQUEST (если запрошена помощь): ждём ответа союзника
5. FREE: предложить сделку, завершить ход
   → Найм доступен через центральную панель поля в любой фазе кроме combat и init_roll
```

## Победа

- Первый до **20 очков** — победа.
- После **120 ходов** — победитель по очкам (тай → по сумме ресурсов).
- Сдача: клетки → нейтральные, армия распускается, ход передаётся.
- Игрок без собственных городов после поражения в бою — выбывает автоматически.

| Действие | Очки |
|--|--|
| Захват источника | +1 |
| Победа в данже (лёгкий) | +1 |
| Победа в данже (средний) | +2 |
| Победа в данже (тяжёлый) | +3 |
| Победа над игроком | +2 |
| Поражение от игрока | −1 |
| Благословение | +2 |

## GameState (актуальные типы)

```typescript
interface GameState {
  board: Cell[];
  players: GamePlayer[];
  currentPlayerIndex: number;
  phase: 'init_roll' | 'roll' | 'cell' | 'joint_request' | 'combat' | 'free' | 'waiting';
  initRolls?: Record<string, number>;
  turnNumber: number;
  hirePool: HireSlot[];
  hireUsed: boolean;
  lastRoll?: number;
  combatData?: CombatData;
  pendingOffer?: TradeOffer;
  pendingEvent?: PendingEvent;
  jointRequest?: JointRequest;
  godMode?: boolean;
  maxTurns: number;   // 120
  winScore: number;   // 20
  log: string[];
}

interface JointRequest {
  from: string;      // id атакующего
  to: string;        // id запрошенного союзника
  cellIndex: number;
  combatType: 'resource' | 'dungeon' | 'city';
}

interface PendingEvent {
  name: string;
  wood?: number;
  stone?: number;
  gold?: number;
  moveDelta?: number;
  deserterUnit?: string | null;
  scoreDelta?: number;
}

interface GamePlayer {
  id: string; name: string; color: string; position: number;
  resources: { wood: number; stone: number; gold: number };
  army: (ArmySlot | null)[];
  artifacts: Artifact[];
  score: number;
  mana: number;           // текущая мана, макс MAX_MANA=10
  hireBonusCount: number; // бонусные найма (накапливаются при остановке в городе)
  surrendered: boolean;
  promises: GamePromise[];
  promisesKept: number;
  promisesBroken: number;
}

interface ArmySlot { unit: Unit; count: number; }
interface HireSlot { unit: Unit; count: number; }

interface Cell {
  index: number;
  name?: string;          // топоним (ресурсные клетки, города)
  type: 'start' | 'resource' | 'dungeon' | 'event';
  resourceType?: 'wood' | 'stone' | 'gold';
  owner?: string;
  garrison?: ArmySlot[];          // нейтральная охрана
  playerGarrison?: (ArmySlot | null)[];  // гарнизон владельца (5 слотов)
  dungeonDifficulty?: 'easy' | 'medium' | 'hard';
  dungeonReward?: DungeonReward;
  dungeonCleared?: boolean;
}

interface Unit {
  id: string; name: string; hp: number; maxHp: number;
  damageMin: number; damageMax: number;
  cost: { wood: number; stone: number; gold: number };
  combatClass: 'melee' | 'ranged';
  universal?: boolean;  // отвечает на атаки любого класса (только Маг)
}

interface Artifact {
  id: string; name: string;
  slot: 'armor' | 'weapon' | 'ring' | 'medallion' | 'extra1' | 'extra2';
  effect: string;
}

interface DungeonReward {
  resources: { wood: number; stone: number; gold: number };
  artifact?: Artifact;
  score: number;
}

interface CombatData {
  events: CombatEvent[];
  initialAtkSnaps: (UnitSnap | null)[];  // 5 слотов (solo) или 10 (joint: 0-4 atk, 5-9 ally)
  initialDefSnaps: UnitSnap[];
  totalRounds: number;
  attackerWon: boolean;
  cellIndex: number;
  combatType: 'resource' | 'dungeon' | 'city';
  attackersLeft: ArmySlot[];
  defendersLeft: ArmySlot[];
  allyPlayerId?: string;
  allyLeft?: ArmySlot[];
}

interface GamePromise {
  type: 'no-attack' | 'help-battle';
  from: string; to: string; turnCreated: number;
}

interface TradeSide {
  wood: number; stone: number; gold: number;
  cellIndices: number[];
  artifactIds: string[];
  promises: PromiseType[];
}

interface TradeOffer {
  id: string; from: string; to: string;
  give: TradeSide;
  want: TradeSide;
}
```

## UI (схема)

**Лобби:** кнопка «Создать», поле кода + имени + «Присоединиться».

**Ожидание:** код комнаты, список игроков, «Начать» (активна при 2–4 игроках).

**Игровой экран:**
```
[Топбар: ход N/30, текущий игрок, последний бросок, ВАШ ХОД]
[Панель игрока z:60] | [ПОЛЕ]                    | [Панель действий z:60]
  Ресурсы 10(+3)       Периметр: клетки            [Торговля]  ← всегда (не combat)

  Артефакты            Города: 🏰 с именами         [✓ Завершить]   ← в free
  Очки ★
  Обещания → / ←
  Клетки (имена)
[Лог событий]
```

Модальные окна (backdrop z:55, box z:65) затемняют только поле — панели остаются активными поверх бэкдропа.

**Бейдж игрока на модалках:** если модалка показывается наблюдателю — отображается имя активного игрока.

## Что НЕ реализовано

- **Походный рюкзак** — эффект «2 найма за ход» не применяется (артефакт есть, логика нет)
- **Обещания** — хранятся и отображаются, но не отслеживаются автоматически (нарушение не штрафуется)
- **Победа над игроком** (+2 / −1) — не начисляется при победе в PvP бою
